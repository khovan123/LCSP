# Manager Assessment Golden Path

## Purpose

This document describes one complete controlled MVP prototype scenario from zero user action to blocked/proceeding downstream workflows. It is implementation guidance only and does not claim validation completion, production readiness, legal reliability or scanner runtime accuracy.

## Golden Path Steps

| Step | User Action | UI Route | HTTP Endpoint | Permission | DTO Input JSON | Controller / Service | Prisma Records Written | RabbitMQ Command/Event | Worker / Module Processing | Expected Output | Audit Event | Failure Behavior |
|---:|---|---|---|---|---|---|---|---|---|---|---|---|
| 1 | Manager signs in with OAuth/OIDC | `/login` -> `/auth/callback` | `POST /api/v1/auth/oauth/callback` | public callback | `{ "code": "oidc-code", "state": "state", "nonce": "nonce" }` | `AuthController.handleOAuthCallback`, `OAuthService.verifyCallback` | `User`, `OAuthIdentity`, `Session` | none | none | `SessionDto` and redirect to dashboard | `audit.auth.login_succeeded.v1` | invalid callback -> 400 `OAUTH_CALLBACK_INVALID` |
| 2 | Manager creates organization and assessment | `/app/assessments/new` | `POST /api/v1/assessments` | `assessment.create` | `{ "organizationId": "uuidv7", "name": "Prototype Assessment" }` | `AssessmentController.create`, `AssessmentService.createAssessment` | `Organization`, `Membership`, `Assessment` | none | none | `AssessmentDto` with `assessmentId` | `audit.assessment.created.v1` | missing Manager membership -> 403 `ORG_MEMBERSHIP_REQUIRED` |
| 3 | Manager completes WizardProfile | `/app/assessments/{assessmentId}/wizard` | `PUT /api/v1/assessments/{assessmentId}/wizard-profile` | `wizard.write` | `{ "businessProcess": "loan review", "declaredAiUse": "decision support" }` | `WizardController.submit`, `WizardService.submitProfile` | `WizardProfile` | none | none | Wizard state `SUBMITTED` | `audit.wizard.submitted.v1` | invalid UUID -> 400 `INVALID_UUID` |

| 8 | AI Usage Flow Builder runs | `/app/assessments/{assessmentId}/ai-usage-flow` | `POST /api/v1/assessments/{assessmentId}/ai-usage-flow` | Manager/system request | `{ "technicalEvidenceReportId": "uuidv7" }` | `AIUsageFlowController.request`, `AIUsageFlowService.requestBuild` | `OutboxEvent` | `command.ai-usage-flow.requested.v1`, `event.ai-usage-flow.completed.v1` | `AIUsageFlowWorker.handleRequested`, `buildAIUsageFlowClaims` | `AIUsageFlow` with evidence-backed claims | `audit.ai_usage_flow.created.v1` | material claim missing evidence -> 422 `AI_USAGE_FLOW_EVIDENCE_REQUIRED` |
| 9 | Reconciliation creates conflict or VerifiedProfile | `/app/assessments/{assessmentId}/reconciliation` | `POST /api/v1/assessments/{assessmentId}/reconciliation` | system/Manager request | `{ "aiUsageFlowId": "uuidv7" }` | `ReconciliationController.request`, `ReconciliationService.requestRun` | `ReconciliationConflict` or `VerifiedProfile` | `command.reconciliation.requested.v1` | `ReconciliationWorker.handleRequested`, `compareWizardAndTechnicalProfile` | conflict list or VerifiedProfile candidate | `audit.reconciliation.conflict_detected.v1` or `audit.verified_profile.created.v1` | material mismatch -> classification blocked |

| 11 | Classification workflow runs only after VerifiedProfile | `/app/assessments/{assessmentId}/classification` | `POST /api/v1/assessments/{assessmentId}/classification` | `classification.request` | `{ "verifiedProfileId": "uuidv7" }` | `ClassificationController.request`, `ClassificationService.requestRun` | `OutboxEvent`, then `RiskClassification` or blocked state | `command.classification.requested.v1`, `event.classification.completed.v1` or `event.classification.blocked.v1` | `ClassificationWorker.handleRequested`, `LegalRetrievalCitationGuardrail` | classification draft or `CLASSIFICATION_BLOCKED` | `audit.classification.completed.v1` or `audit.classification.blocked.v1` | missing citation -> 422 `CITATION_REQUIRED` |
| 12 | Gap/document workflow remains blocked or proceeds | `/app/assessments/{assessmentId}/documents` | `POST /api/v1/assessments/{assessmentId}/documents` | `document.request` | `{ "classificationId": "uuidv7", "documentType": "prototype_summary" }` | `DocumentController.request`, `DocumentService.requestGeneration` | `GeneratedDocument` metadata if allowed | `command.document.requested.v1`, `event.document.generated.v1` | `DocumentWorker.handleRequested` | generated artifact ref with prototype warning or blocked state | `audit.document.generated.v1` or `audit.document.blocked.v1` | unresolved conflict/citation -> `DOCUMENT_GENERATION_BLOCKED` |

## User Observation Model

The Manager observes progress through dashboard status cards. Each card maps to DB state and audit events, not optimistic UI state alone. A blocked state must say why the workflow is blocked: missing evidence, unresolved conflict, missing citation, insufficient quality or unsupported analysis boundary.
